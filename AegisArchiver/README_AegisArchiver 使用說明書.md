# 📂 Script_AegisArchiver (pdf 去毒歸檔)
## 掃描 PDF 惡意內容，風險檔案隔離、安全檔案去毒後歸檔

AegisArchiver 是一套具備溯源追蹤能力與自動化資安防護的檔案歸檔工具。本程式設計用於批次處理指定目錄下的靜態檔案，能自動偵測 PDF 檔案的潛在風險標籤並進行實體隔離，確保歸檔區（Safe Zone）的檔案純淨，並產出符合單一真相來源（SSOT）標準的審計報表。

## 核心機制

* **深層 PDF 檢測與去毒**：自動掃描 PDF 內的 `/JS`、`/JavaScript`、`/Launch`、`/OpenAction` 等高風險標籤。安全的 PDF 將自動拔除外部連結（`/A`, `/AA`），風險 PDF 則阻斷進入安全區。
* **全域 I/O 異常攔截**：具備檔案鎖定與狀態變更的防禦機制，任何檔案讀寫失敗皆會被記錄，保證主掃描迴圈與報表產出不中斷。
* **嚴格的實體隔離與驗證**：高風險原檔移至隔離區前，必須經過 SHA-256 雜湊驗證，確保原始資料 100% 完整複製後才會刪除原檔。
* **原子級報表寫入**：審計報表（CSV）採用暫存檔替換技術（Atomic Swap），防止執行中斷導致報表毀損。

## 🛠 系統環境與建置

### 1. 系統需求

* **作業系統**：Windows / macOS / Linux
* **Python 版本**：Python 3.8 或以上版本

### 2. 安裝依賴套件

本工具依賴 `pikepdf` 進行底層 PDF 解析，以及 `tqdm` 產生進度條。請於終端機執行以下指令安裝：

```bash
pip install pikepdf tqdm

```

---

## 🚀 使用方法

### 1. 執行程式

在終端機中執行腳本：

```bash
python Script_AegisArchiver.py

```

### 2. 輸入目標路徑

程式啟動後，會提示輸入需要掃描的原始資料夾路徑。支援絕對路徑與相對路徑（可直接拖曳資料夾至終端機視窗中）：

```text
Enter source directory path: C:\Path\To\Your\Archive\Files

```

### 3. 執行過程與輸出

程式會自動遞迴掃描路徑下的目標檔案（`.pdf`, `.txt`, `.jpg`, `.jpeg`, `.png`, `.gif`, `.xlsx`, `.docx`, `.pptx`），並於原始資料夾的**上一層目錄**建立安全歸檔區與記錄檔。

#### 目錄結構說明

執行完畢後，將產生以下結構：

* `SafeZone_Archived_[時間戳記]/`：**安全區**。存放乾淨的原檔與去毒後的 PDF。若為被隔離的風險檔案，此處會留下 `.quarantined.txt` 溯源佔位符。
* `00_Security_Audit_Report.csv`：**審計報表 (SSOT)**，記錄所有檔案的來源、大小、狀態與最終去向。
* `00_Execution_Log.log`：**系統日誌**，記錄執行過程中的詳細 [INFO]、[WARN]、[ERROR] 與 [CRIT] 狀態。


* `[原始目錄]/!_Infected_Originals/`：**隔離區**。存放被判定為具備惡意腳本或高風險操作的 PDF 原始檔案。
* `[原始目錄]/!_Failed_Processing/`：**失敗備份區**。存放因檔案損毀或未預期錯誤而無法正常解析的檔案。

## ⚠️ 限制與注意事項

為確保系統使用的預期心理正確，請詳閱以下邊界限制：

1. **非防毒軟體替代品**
本腳本對 PDF 的檢測基於「靜態字典特徵匹配」（尋找特定的字典標籤）。它無法解析經過高度混淆（Obfuscation）的惡意程式碼，也無法防禦利用 PDF 解析器漏洞（Zero-day exploits）的攻擊。
2. **非 PDF 檔案的掃描盲區**
對於 `.docx`、`.xlsx` 等 Office 檔案，或 `.jpg` 等圖片檔，本程式**僅執行安全的檔案複製**，不會進行任何巨集（Macro）檢測、格式解析或去毒操作。若原始 Office 檔案帶有惡意巨集，歸檔至安全區的檔案依然會帶有該巨集。
3. **儲存空間需求**
為符合不覆寫原始檔案的原則，本工具採用「複製後處理」的策略（除被隔離的風險檔案外，原檔皆會保留）。因此，執行前請確保硬碟具備**至少等同於原始資料夾總大小**的額外可用空間。
4. **檔案鎖定處理 (File Locking)**
若掃描過程中，檔案正被其他應用程式（如 Word、PDF 閱讀器或防毒軟體）開啟或鎖定，程式將無法對其進行處理。這類檔案會被自動跳過，並於報表中標記為 `[LOCKED_OR_LOST_PRE_PROCESS]` 或 `[LOCKED_OR_UNREADABLE]`，**不會**被搬移至安全區。
